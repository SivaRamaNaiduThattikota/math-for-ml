Section 1Information is surprise

How much do you learn from an event? If it was certain, nothing; if it was a shock, a lot. Information theory makes this precise: the information content (or "surprise") of an outcome with probability $p(x)$ is

$$ I(x) = -\log p(x) $$

A near-certain event ($p\to1$) carries $\approx 0$ information; a rare one ($p\to0$) carries a lot. The log makes information additive for independent events (their probabilities multiply, so their surprises add) — and measured with $\log_2$, it's literally the number of bits needed to encode the outcome.

Your Power BI / SQL anchor "The server is up" — an expected, low-information row. "The server is on fire" — rare, high-information: it's the row that changes your day. Information theory quantifies that gut sense of "how newsworthy is this value," and it turns out to be exactly the loss your classifiers minimize.

Section 2Entropy — a distribution's uncertainty

Entropy is the average surprise of a distribution — the expected information per draw:

$$ H(p) = -\sum_x p(x)\log p(x) = \mathbb{E}[-\log p(x)] $$

It measures uncertainty. A uniform distribution (all outcomes equally likely) has maximum entropy — you can't predict anything. A peaked distribution (one outcome almost certain) has entropy near 0 — barely any surprise. Slide the true probability below and watch the binary entropy peak at $p=0.5$ (1 bit — a fair coin, maximally uncertain) and fall to 0 at the extremes.

Section 3Cross-entropy — the cost of a wrong model

Now suppose reality follows $p$ but your model believes $q$. Your average surprise, scored against the true frequencies, is the cross-entropy:

$$ H(p,q) = -\sum_x p(x)\log q(x) $$

It is always at least the entropy, $H(p,q)\ge H(p)$, with equality only when $q=p$ — you can never be less surprised than reality's inherent uncertainty, and a wrong model makes you more surprised. This is exactly the classification loss from Session 17: the true distribution $p$ is the one-hot label, $q$ is the model's predicted probabilities, and $H(p,q)=-\log q(\text{true class})$. Minimizing cross-entropy trains the model's beliefs $q$ toward the truth $p$.

Section 4KL divergence — how far apart two distributions are

The Kullback–Leibler divergence is the extra surprise from using $q$ instead of the true $p$ — the gap between cross-entropy and entropy:

$$ D_{\mathrm{KL}}(p\,\|\,q)=\sum_x p(x)\log\frac{p(x)}{q(x)} = \underbrace{H(p,q)}_{\text{cross-entropy}}-\underbrace{H(p)}_{\text{entropy}} $$

It's a "distance" between distributions: $D_{\mathrm{KL}}\ge 0$ always, and $=0$ exactly when $q=p$. Rearranged, cross-entropy = entropy + KL. Since the true distribution's entropy $H(p)$ is fixed, minimizing cross-entropy is minimizing the KL divergence — pulling the model $q$ onto the data distribution $p$.

Going deeper — KL is not symmetric (and why that matters)

Crucially $D_{\mathrm{KL}}(p\|q)\ne D_{\mathrm{KL}}(q\|p)$ — it's a divergence, not a true distance (no symmetry, no triangle inequality). The direction changes behaviour: forward KL $D(p\|q)$ is "mode-covering" (q spreads to cover all of p's mass, penalizing $q\to0$ where $p>0$), while reverse KL $D(q\|p)$ is "mode-seeking" (q locks onto one mode). This choice shapes VAEs (which minimize a reverse-KL-flavoured term), distillation, and variational inference. And when $p$ is one-hot, $H(p)=0$, so cross-entropy equals KL equals $-\log q(\text{true class})$ — the classification loss is literally a KL divergence to the label. Beyond MML core; used throughout its ML chapters.

Section 5In NumPy ✓ outputs verified

import numpy as np

def entropy(p):        return -np.sum(p*np.log2(p + 1e-12))
def cross_entropy(p, q): return -np.sum(p*np.log2(q + 1e-12))
def kl(p, q):          return np.sum(p*np.log2((p+1e-12)/(q+1e-12)))

p = np.array([0.7, 0.2, 0.1])          # truth
q = np.array([0.5, 0.3, 0.2])          # model
print(round(entropy(p),3))              # 1.157 bits
print(round(cross_entropy(p,q),3))      # 1.28 bits (> entropy)
print(round(kl(p,q),3))                   # 0.123  = CE - H

# cross-entropy = entropy + KL, and KL >= 0
print(np.isclose(cross_entropy(p,q), entropy(p)+kl(p,q)))   # True

# one-hot label -> cross-entropy is just -log q(true class)
print(round(-np.log2(0.5), 3))         # 1.0 bit if true class had q=0.5
1.157 1.28 0.123 True 1.0
Add an epsilon, mind the base $\log 0=-\infty$, so clamp with a tiny $\varepsilon$ (or use log_softmax/logits). Units depend on the log base: $\log_2$ → bits, $\ln$ → nats (what deep-learning frameworks use). The shape of the loss is identical; only the scale differs.

Section 6Where it lives in real ML

  • Cross-entropy loss — the default for classification and language modelling (next-token prediction is cross-entropy over the vocabulary).
  • VAEs — the training objective (ELBO) has a KL term pulling the latent posterior toward a prior.
  • Knowledge distillation — train a small "student" to match a big "teacher" by minimizing KL between their output distributions.
  • RLHF / PPO — a KL penalty keeps the fine-tuned LLM policy close to the reference model, preventing it from drifting into gibberish.
  • Mutual information $I(X;Y)$ (a KL between the joint and the product of marginals) drives feature selection and representation learning; t-SNE minimizes a KL; label smoothing softens the one-hot $p$.
The one-line takeaway Information is surprise ($-\log p$); entropy is average surprise; cross-entropy is your surprise under a wrong model; KL is the excess. Cross-entropy = entropy + KL, so minimizing the classification loss pulls your model's distribution onto the truth. It's the through-line from probability to nearly every modern loss.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • $\log 0$ — a model assigning probability 0 to something that occurs gives infinite loss; clamp or use logits.
  • KL is asymmetric — $D(p\|q)\ne D(q\|p)$; pick the direction deliberately (mode-covering vs mode-seeking).
  • KL requires $q>0$ wherever $p>0$ — otherwise it's infinite; forward KL punishes a model that rules out possible outcomes.
  • Bits vs nats — $\log_2$ vs $\ln$ only rescales by $\ln 2$; don't compare numbers across bases.

Golden rules

  1. Information $=-\log p$: rarer = more surprising = more bits.
  2. Entropy $H(p)=-\sum p\log p$: average surprise; max at uniform, 0 at certain.
  3. Cross-entropy $H(p,q)=-\sum p\log q \ge H(p)$: cost of model $q$ under truth $p$.
  4. KL $=H(p,q)-H(p)\ge0$, $=0$ iff $q=p$; asymmetric.
  5. Minimizing cross-entropy (fixed $p$) = minimizing KL(p‖q) = fitting the model to the data distribution.
Trap 1 · Treating KL as a symmetric distance $D(p\|q)\ne D(q\|p)$ and there's no triangle inequality. It's a divergence — the direction matters and encodes different fitting behaviour.
Trap 2 · Zeros in the model If $q(x)=0$ where $p(x)>0$, cross-entropy and KL blow up to infinity. Never let a model assign exactly 0 to a possible class — softmax + smoothing/logits avoid it.
Trap 3 · Confusing bits and nats Frameworks report cross-entropy in nats ($\ln$); textbooks often use bits ($\log_2$). Same optimum, different number — don't compare across the two.

InterviewQuestions you should be able to answer

Q1 · What is entropy, intuitively?
The average surprise (information) of a distribution — its uncertainty. Maximal for a uniform distribution, zero for a deterministic one; measured in bits ($\log_2$) or nats ($\ln$).
Q2 · What does cross-entropy measure, and why is it the classification loss?
The average surprise of scoring data (true $p$) with a model ($q$): $-\sum p\log q$. With one-hot labels it's $-\log q(\text{true class})$, so minimizing it maximizes the probability the model assigns to the correct answers.
Q3 · Define KL divergence and its key properties.
$D(p\|q)=\sum p\log(p/q)=H(p,q)-H(p)$. It's $\ge0$, zero iff $p=q$, and asymmetric (not a metric). It's the extra bits from using $q$ instead of $p$.
Q4 · Why is minimizing cross-entropy the same as minimizing KL?
Cross-entropy = entropy(p) + KL(p‖q); with fixed true labels, entropy(p) is constant, so minimizing cross-entropy minimizes KL — fitting the model distribution to the data.
Q5 · Where does KL show up beyond classification?
VAE ELBO (latent vs prior), knowledge distillation (student vs teacher), RLHF PPO (policy vs reference), mutual information, and t-SNE.
