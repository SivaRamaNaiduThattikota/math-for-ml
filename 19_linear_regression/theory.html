Linear regression & bias–variance

The template model: predict with a dot product, fit by least squares (= Gaussian MLE), judge with R² and residuals, and balance bias against variance.

Section 1The linear model 📖 MML Ch 9

Linear regression predicts a number as a weighted sum of features — a dot product (Session 2) plus a bias:

$$ \hat y = w_1x_1 + \dots + w_dx_d + b = \mathbf{w}^\top\mathbf{x}+b $$

Fold the bias into the weights by adding a constant "1" feature, so the whole dataset is a single matrix equation $\hat{\mathbf{y}}=X\mathbf{w}$ (Session 4). Fitting means choosing $\mathbf{w}$ so the predictions are close to the true $\mathbf{y}$ — and "close" is measured by squared error.

Your Power BI / SQL anchor It's the trend line you've dropped on countless charts — but multivariable and principled. Instead of one predictor, you weight many columns; instead of eyeballing, you solve for the weights that minimize total squared error. Every "what drives revenue?" coefficient question is linear regression.

Section 2Least squares & the normal equation

We pick $\mathbf{w}$ to minimize the sum of squared residuals $\sum_i(y_i-\hat y_i)^2$. Setting the gradient to zero (Session 9) gives a closed-form solution — the normal equation:

$$ \hat{\mathbf{w}} = (X^\top X)^{-1}X^\top\mathbf{y} $$

Notice $X^\top X$ is (a scalar multiple of) the covariance matrix (Session 14) — and it's invertible only if the features are independent (full rank, Session 5). Collinear features make it singular, which is why multicollinearity breaks regression. Crucially, minimizing squared error is exactly maximum likelihood under Gaussian noise (Session 17): least squares isn't arbitrary, it's the MLE for $y=\mathbf{w}^\top\mathbf{x}+\varepsilon,\ \varepsilon\sim\mathcal{N}(0,\sigma^2)$.

Section 3$R^2$ & residuals — is the fit any good?

The coefficient of determination $R^2$ measures the fraction of the target's variance the model explains:

$$ R^2 = 1 - \frac{\sum_i(y_i-\hat y_i)^2}{\sum_i(y_i-\bar y)^2} = 1 - \frac{\text{SS}_{\text{res}}}{\text{SS}_{\text{tot}}} $$

$R^2=1$ is a perfect fit; $R^2=0$ is no better than predicting the mean; it can even go negative on test data for a bad model. But a single number hides a lot — always look at the residuals $r_i=y_i-\hat y_i$. Linear regression assumes they are random, centered at zero, equal-variance (homoscedastic), and roughly normal. A residual plot (residuals vs predictions) should look like formless noise; any pattern — a curve, a fan shape, clusters — means an assumption is violated (nonlinearity, heteroscedasticity, missing feature).

R² alone lies A high $R^2$ can hide curved residuals (wrong model), and $R^2$ always increases as you add features even useless ones (use adjusted $R^2$). Judge a regression by its residual plot and its test performance, not the training $R^2$.

Section 4The bias–variance trade-off 📖 MML Ch 8

Why not just fit a wiggly high-degree curve that nails every point? Because a model's expected test error splits into two competing pieces:

$$ \text{Error} = \underbrace{\text{Bias}^2}_{\text{too simple}} + \underbrace{\text{Variance}}_{\text{too sensitive}} + \underbrace{\text{irreducible noise}}_{\text{unavoidable}} $$
  • High bias (underfitting) — the model is too simple (a line for a curve); it misses the pattern on both train and test.
  • High variance (overfitting) — the model is too flexible; it memorizes noise, so train error is tiny but test error explodes.

The sweet spot is in between. Drag the polynomial degree below and watch it happen — the fitted curve go from too-stiff to just-right to wildly overfit, with the test error tracing the classic U-shape.

Section 5In NumPy ✓ outputs verified

import numpy as np
rng = np.random.default_rng(0)
n = 100
x = rng.uniform(0, 5, n)
y = 2.5*x + 1.0 + rng.normal(0, 1.5, n)   # true slope 2.5, intercept 1

X = np.column_stack([np.ones(n), x])         # design matrix [1, x]
w = np.linalg.lstsq(X, y, rcond=None)[0]      # least squares
print(w.round(3))                            # [0.989 2.461]  ~ intercept 1, slope 2.5

yhat = X @ w
r2 = 1 - np.sum((y-yhat)**2)/np.sum((y-y.mean())**2)
print(round(r2, 3))                          # 0.868

# the normal equation gives the same weights
print(np.allclose(w, np.linalg.inv(X.T@X) @ X.T @ y))   # True
[0.989 2.461] 0.868 True

Section 6Where it lives in real ML

  • The template for supervised learning. Logistic regression, neural nets, and more all generalize this: a parameterized prediction, a likelihood-derived loss, fit by optimization.
  • Regularization = variance control. Ridge adds $\lambda\lVert\mathbf{w}\rVert_2^2$ (an L2 penalty = a Gaussian prior, Session 17) to shrink weights and cut overfitting; lasso (L1) also selects features. The $\lambda$ slides you along the bias–variance curve.
  • Cross-validation finds the sweet-spot complexity by estimating test error — directly managing bias vs variance.
  • Diagnostics transfer — residual analysis, $R^2$, and multicollinearity checks (Session 5) are everyday modelling tools.
The one-line takeaway Linear regression = predict with a dot product, fit by least squares (= Gaussian MLE), judge with $R^2$ and residuals, and control the bias–variance trade-off with model complexity and regularization. It's the whole ML pipeline in miniature.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Collinear features make $X^\top X$ singular → no unique solution; use ridge or drop features (Session 5).
  • $R^2$ grows with more features regardless of usefulness — use adjusted $R^2$ or test-set $R^2$.
  • Outliers dominate squared error; a few points can swing the whole line (consider robust/Huber loss).
  • Extrapolation is dangerous — a linear fit says nothing reliable beyond the data's range.

Golden rules

  1. Prediction $=\mathbf{w}^\top\mathbf{x}+b$; fit by minimizing squared error (= Gaussian MLE).
  2. Closed form: $\hat{\mathbf{w}}=(X^\top X)^{-1}X^\top\mathbf{y}$ — needs full-rank features.
  3. $R^2$ = variance explained; always check the residual plot too.
  4. Bias–variance: too simple underfits, too flexible overfits; error is a U-curve in complexity.
  5. Regularization (ridge/lasso) trades a little bias for a lot less variance.
Trap 1 · Trusting training R² Complex models always fit training data better. A great train $R^2$ with poor test performance is overfitting — evaluate on held-out data / cross-validation.
Trap 2 · Ignoring residual patterns Curved or fan-shaped residuals mean the linear+Gaussian assumptions are wrong — the coefficients and p-values are then untrustworthy. Plot residuals every time.
Trap 3 · Multicollinearity Highly correlated predictors make coefficients unstable and $X^\top X$ near-singular. Check correlations/VIF; regularize or drop.

def ridge(X,y,lam): return np.linalg.solve(X.T@X+lam*np.eye(X.shape[1]), X.T@y)

Small $\lambda$ barely changes the overfit; moderate $\lambda$ tames the wiggles and cuts test error; huge $\lambda$ over-shrinks toward a flat line (back to high bias). $\lambda$ is the bias–variance dial.

InterviewQuestions you should be able to answer

Q1 · How is linear regression fit, and how does it relate to MLE?
By least squares — minimizing the sum of squared residuals, with closed form $\hat w=(X^\top X)^{-1}X^\top y$. This is exactly maximum likelihood assuming Gaussian noise on the targets.
Q2 · What does R² mean, and its limitations?
The fraction of target variance explained (1 = perfect, 0 = no better than the mean, can be negative on test data). It always rises with more features, so use adjusted R² and always inspect residuals and test performance.
Q3 · Explain the bias–variance trade-off.
Expected error = bias² + variance + irreducible noise. Simple models have high bias (underfit); flexible models have high variance (overfit). Optimal complexity minimizes their sum; regularization and cross-validation manage it.
Q4 · What assumptions does linear regression make, and how do you check them?
Linearity, independent errors, constant variance (homoscedasticity), and roughly normal residuals. Check with a residual-vs-fitted plot (should be patternless), a QQ-plot, and multicollinearity diagnostics.
Q5 · How does ridge regression help?
It adds an L2 penalty (a Gaussian prior on weights, MAP), shrinking coefficients to reduce variance/overfitting and stabilizing $X^\top X$ under collinearity — trading a little bias for much less variance.
