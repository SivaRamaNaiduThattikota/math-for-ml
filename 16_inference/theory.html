Inference & hypothesis testing

Is this effect real or just noise? Confidence intervals, p-values, and the A/B test that ships product and model decisions.

~22 min readNumPy paired notebookPrereq: Sampling & the CLT

You shipped a new model and accuracy went from 91.2% to 91.6%. Real improvement, or luck of the test set? Your redesigned checkout converted 4.1% vs 3.8% — launch it, or noise? Statistical inference answers exactly these questions: it uses the sampling distribution (Session 15) to decide whether an observed effect is real or could easily be random wobble. Confidence intervals, p-values, and the A/B test are its tools — and they decide product launches and model rollouts every day.

This is the most directly job-relevant session in the probability pillar: A/B testing and "is this difference significant?" come up in nearly every data-science interview.

By the end you will be able to
  • Build and correctly interpret a confidence interval.
  • Run a hypothesis test: null vs alternative, test statistic, p-value, significance level $\alpha$.
  • Distinguish Type I / Type II errors and understand statistical power.
  • Set up and read an A/B test of two conversion rates.
  • Avoid the classic pitfalls: p-hacking, multiple comparisons, and significance ≠ importance.

Section 1Confidence intervals

A point estimate ($\bar{X}=3.8\%$) hides its uncertainty. A confidence interval attaches a range, using the standard error from Session 15:

$$ \text{estimate} \;\pm\; z^{*}\cdot \text{SE} \qquad (z^{*}=1.96 \text{ for } 95\%) $$

The correct interpretation is subtle and a favourite interview trap: a 95% CI means "if we repeated the experiment many times, 95% of the intervals we build would contain the true value." It is not "95% probability the true value is in this interval" — the true value is fixed; the interval is what's random. Practically: wider CI = more uncertainty; it shrinks as $\sqrt{n}$ grows.

Your Power BI / SQL anchor It's the error bar you wish every dashboard KPI had. "Conversion = 3.8%" is a point; "3.8% ± 0.4%" tells you whether last week's 4.1% is a real move or within noise. Inference is the discipline of drawing those error bars honestly.

Section 2Hypothesis testing & the p-value

A hypothesis test is a formal skeptic. You state a null hypothesis $H_0$ ("no effect — the two versions convert equally") and an alternative $H_1$ ("there is a difference"). Then you ask: if $H_0$ were true, how surprising is the data I saw? That surprise is the p-value:

$$ p = P(\text{observing data this extreme, or more}\ \mid\ H_0\ \text{is true}) $$

Small $p$ = the data would be very unlikely under "no effect," so you reject $H_0$ and call the result statistically significant. The threshold is the significance level $\alpha$ (commonly 0.05): reject when $p<\alpha$. The recipe: compute a test statistic (how many standard errors the effect is from zero), convert it to a p-value via the sampling distribution (Gaussian, by the CLT).

Section 3Type I / Type II errors & power

A test can be wrong two ways:

  • Type I error (false positive) — reject $H_0$ when it's actually true (declare an effect that isn't real). Its rate is $\alpha$.
  • Type II error (false negative) — fail to reject $H_0$ when there is a real effect (miss it). Its rate is $\beta$.

Power $=1-\beta$ is the probability of detecting a real effect. Power rises with larger sample size, bigger true effect, and higher $\alpha$ (though raising $\alpha$ trades away false-positive control). Underpowered tests waste effort — they can't see the effect even if it's there — which is why you compute the needed $n$ before running an experiment.

Section 4The A/B test

The canonical application: split users into control (A) and treatment (B), measure a metric (say conversion rate), and test whether B truly differs from A. $H_0$: the rates are equal. Simulate one below — adjust the true rates and sample size, and re-run experiments to feel how randomness produces "significant" results that come and go when the effect is small or $n$ is low.

Section 5Pitfalls that ruin conclusions

  • Misreading the p-value — it is not the probability that $H_0$ is true, nor the probability your result is a fluke. It's P(data | $H_0$).
  • Significance ≠ importance — with huge $n$, a trivial 0.01% lift can be "significant." Always report the effect size and CI, not just $p<0.05$.
  • Multiple comparisons — test 20 features at $\alpha=0.05$ and you expect ~1 false positive by chance. Correct for it (Bonferroni, FDR).
  • Peeking / early stopping — checking an A/B test repeatedly and stopping when it's "significant" massively inflates false positives.
Going deeper — what a p-value is not

The p-value is $P(\text{data}\mid H_0)$, not $P(H_0\mid \text{data})$ — flipping those is the Bayesian/base-rate confusion from Session 12. A p-value of 0.03 does not mean "3% chance the null is true" or "97% chance my effect is real"; it means "if there were truly no effect, data this extreme would occur 3% of the time." Likewise a 95% CI is a statement about the procedure's long-run coverage, not the probability for one specific interval. And "fail to reject $H_0$" is not "prove $H_0$" — absence of evidence isn't evidence of absence, especially in an underpowered test. These distinctions are exactly the interview traps. Section 6In NumPy — and where it lives in ML ✓ outputs verified

import numpy as np
from math import erf, sqrt
Phi = lambda z: 0.5*(1 + erf(z/sqrt(2)))          # standard-normal CDF

# A/B test: two-proportion z-test
nA, nB = 2000, 2000
cA, cB = 200, 240                              # conversions
pA, pB = cA/nA, cB/nB
p = (cA+cB)/(nA+nB)                              # pooled rate under H0
se = sqrt(p*(1-p)*(1/nA + 1/nB))
z  = (pB - pA)/se
pval = 2*(1 - Phi(abs(z)))                     # two-sided
print(round(pB-pA,3), round(z,2), round(pval,4))     # lift, z, p-value

# 95% CI for the difference
se_d = sqrt(pA*(1-pA)/nA + pB*(1-pB)/nB)
lo, hi = (pB-pA) - 1.96*se_d, (pB-pA) + 1.96*se_d
print(round(lo,3), round(hi,3))                  # CI excludes 0 -> significant
0.02 2.02 0.0432 0.001 0.039 # 95% CI for the lift excludes 0
  • Model comparison — "is model B's higher accuracy real?" is a hypothesis test / CI on the metric difference (often via a paired test or bootstrap CI).
  • Feature & launch decisions — A/B tests gate product and model rollouts; power analysis sets the sample size up front.
  • Multiple-testing correction — trying many models/features/hyperparameters inflates false positives; correct or use a held-out final test.
  • Error bars on metrics — bootstrap CIs (Session 15) turn a single accuracy number into an honest range.
The one-line takeaway Inference turns "we saw a difference" into "the difference is (un)likely to be noise." Confidence intervals give a range; p-values quantify surprise under "no effect"; A/B tests apply it to decisions — just don't misread the p-value or ignore effect size.
Computational & memory cost A test statistic is $O(n)$ to compute; the real cost is the sample size needed for adequate power — it grows as the effect you want to detect shrinks (roughly $\propto1/\text{effect}^2$). And testing many hypotheses inflates false positives, requiring correction.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • p-value ≠ P(H₀ true) — it's P(data | H₀). The flip requires a prior (Bayes, Session 12).
  • Not significant ≠ no effect — could be an underpowered test; report the CI, which shows what effect sizes remain plausible.
  • Big n makes tiny effects significant — significance is about detectability, not importance; report effect size.
  • Assumptions: tests assume independent samples and (via CLT) enough data; correlated or tiny samples break them.

Golden rules

  1. CI = estimate ± $z^{*}\cdot$SE; a 95% CI covers the truth 95% of the time across repeats.
  2. p-value = P(data this extreme | $H_0$); reject $H_0$ when $p<\alpha$.
  3. Type I = false positive ($\alpha$); Type II = false negative ($\beta$); power $=1-\beta$.
  4. A/B test: two-sample test of a difference; power grows with $n$ and effect size.
  5. Always report effect size + CI, correct for multiple comparisons, and don't peek.
Trap 1 · Misinterpreting the p-value "p=0.03 → 97% chance my effect is real" is wrong. It's P(data|no effect)=3%. Confusing it with P(hypothesis|data) is the base-rate error from Session 12.
Trap 2 · Significance without effect size With enough data everything is significant. A launch decision needs the size of the lift and its CI, not just $p<0.05$.
Trap 3 · Peeking & multiple comparisons Repeatedly checking a running A/B test, or testing many variants, inflates false positives fast. Fix the sample size in advance and correct for multiplicity (Bonferroni/FDR).

Quick self-check

A test gives p = 0.03. Which statement is correct?
Your A/B test isn't significant, but the CI for the lift is [−1%, +9%]. Best conclusion?

PracticeExercises

Hints only — solve in the paired practice.ipynb.

E1easy

A sample of 400 has mean 50 and std 10. Build the 95% CI for the mean.

Show solution

SE $=10/\sqrt{400}=0.5$; CI $=50\pm1.96(0.5)=[49.02,\ 50.98]$.

E2easy

A/B: A = 200/2000, B = 240/2000. Compute the lift and the two-proportion z-statistic.

Show solution

Lift $=0.12-0.10=0.02$; pooled $p=0.11$, SE $=\sqrt{0.11\cdot0.89(1/2000+1/2000)}\approx0.0099$; $z\approx2.02$ → $p\approx0.043$, significant at 0.05.

E3medium

Write a two-sided p-value function from the normal CDF and reproduce E2's p-value.

Show solution
from math import erf, sqrt
Phi=lambda z:0.5*(1+erf(z/sqrt(2)))
z=2.02; 2*(1-Phi(abs(z)))   # ~0.043
E4medium

Simulate the false-positive rate: run 10,000 A/B tests with no true effect ($p_A=p_B=0.1$, $n=1000$) and count how often $p<0.05$. What should it be?

Show solution

About 5% — that's the definition of $\alpha$: with a true null, you falsely "detect" an effect 5% of the time. This is why multiple testing needs correction.

E5hard

Power: with $p_A=0.10$, $p_B=0.12$, estimate the power at $n=1000$ vs $n=5000$ by simulating many experiments and measuring the fraction that reach $p<0.05$.

Show solution

Power is much higher at $n=5000$ than $n=1000$ — a real 2-point lift is detected far more reliably with more data. Simulate: draw binomials, run the z-test, and average the "significant" flag over many runs.

InterviewQuestions you should be able to answer

Q1 · What exactly is a p-value?
The probability of observing data at least as extreme as yours if the null hypothesis were true. Small p → the data is surprising under "no effect" → reject the null. It is not P(H₀ true).
Q2 · How do you interpret a 95% confidence interval?
Across many repeated experiments, 95% of the intervals constructed this way would contain the true parameter. For a single interval you don't assign it a 95% probability — the parameter is fixed.
Q3 · Type I vs Type II error, and power?
Type I = false positive (reject a true null, rate α). Type II = false negative (miss a real effect, rate β). Power = 1−β, the chance of detecting a true effect; it grows with n and effect size.
Q4 · How would you run an A/B test?
Randomize users to control/treatment, fix the metric and sample size (power analysis) in advance, run to completion (no peeking), then test the difference (e.g. two-proportion z-test) and report the effect size with a CI.
Q5 · Why can a statistically significant result be unimportant?
With large n, even a trivially small effect yields a small p-value. Significance measures detectability, not magnitude — always report effect size and CI.
