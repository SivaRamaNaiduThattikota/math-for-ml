Matrix multiplication

Multiplying two matrices means doing one transformation after another. The row-times-column rule is just the bookkeeping of composition.

~20 min readNumPy paired notebookPrereq: Matrices, Dot product

You now know a matrix is a transformation of space (Session 3). So here's the question that makes matrix multiplication obvious instead of arbitrary: if a matrix is a transformation, what is multiplying two of them? The answer is beautifully simple — do one transformation, then the other. Matrix multiplication is composition. The infamous "row-times-column" rule isn't a definition to memorize; it's the bookkeeping that falls out of "apply B, then apply A."

This is the operation a GPU spends almost all its time on. A neural network's forward pass is a chain of matrix multiplications; attention is matrix multiplications; a batch of 10,000 predictions is one matrix multiplication. Understand it as composition and the rest of deep learning is reading.

By the end you will be able to
  • Read $AB$ as composition — "do $B$ first, then $A$" — not just a number-crunching rule.
  • Compute a product two ways: entry-by-entry as row · column dot products, and column-by-column.
  • Explain why matrix multiplication is not commutative ($AB \ne BA$) but is associative.
  • Apply the shape rule $(m\times n)(n\times p) = m\times p$ and spot invalid products instantly.
  • See a neural-network forward pass and attention as chains of matrix products.

Notation reference
$AB$Matrix product — apply B first, then A (right-to-left). $(AB)_{ij}$Entry in row i, col j = (row i of A) · (col j of B). $m\times n$A matrix with m rows, n columns; maps $\mathbb{R}^n\to\mathbb{R}^m$. $I$Identity matrix: $AI = IA = A$ (the "do nothing" transform). $A^\top$Transpose — rows become columns (appears in attention as $QK^\top$).

Section 1Multiplication is composition 📖 MML §2.7

Take two transformations: $B$ (say, a shear) and $A$ (say, a rotation). Apply $B$ to a vector, then apply $A$ to the result: $A(B\mathbf{x})$. Because both are linear, the combined operation is itself a single linear transformation — and the matrix that performs it in one step is written $AB$:

$$ A(B\mathbf{x}) = (AB)\,\mathbf{x} $$

Read $AB$ right-to-left: the matrix nearest the vector acts first. This ordering feels backwards at first, but it's forced by the notation $A(B\mathbf{x})$ — $B$ touches $\mathbf{x}$ first. It's the same reason $f(g(x))$ means "$g$ first."

Your Power BI / SQL anchor Think of a Power Query pipeline: "remove columns → then group → then sort." Each step transforms the table; the pipeline is their composition. Matrix multiplication is exactly that for linear transformations — AB is a two-step pipeline collapsed into one reusable transformation. And just like query steps, order changes the result.

Section 2The row · column rule — and why it's that 📖 MML §2.2

To compute the product, each entry of $AB$ is a dot product of a row of $A$ with a column of $B$ — Session 2 returning with a job:

$$ (AB)_{ij} = (\text{row } i \text{ of } A)\cdot(\text{column } j \text{ of } B) = \sum_k A_{ik}B_{kj} $$

A concrete 2×2:

$$ \begin{bmatrix}1&2\\0&1\end{bmatrix}\begin{bmatrix}2&0\\1&3\end{bmatrix} = \begin{bmatrix} 1\cdot2+2\cdot1 & 1\cdot0+2\cdot3 \\ 0\cdot2+1\cdot1 & 0\cdot0+1\cdot3 \end{bmatrix} = \begin{bmatrix}4&6\\1&3\end{bmatrix} $$
Column viewThere's a cleaner way to see it: column j of AB is A applied to column j of B. B says where the basis vectors go; A then transforms those landing spots. Composition, column by column.
Going deeper — why "row × column" is the same as composition

Column $j$ of $AB$ is, by definition, $(AB)$ applied to the basis vector $\mathbf{e}_j$. But $(AB)\mathbf{e}_j = A(B\mathbf{e}_j)$, and $B\mathbf{e}_j$ is just column $j$ of $B$ (Session 3: columns are basis images). So column $j$ of $AB$ = $A$ times column $j$ of $B$. Working that out entry-by-entry — each output entry is a row of $A$ dotted with that column of $B$ — is the row×column rule. The rule isn't arbitrary; it's the only bookkeeping consistent with "apply B, then A." See MML §2.2–2.7.

Section 3Order matters: $AB \ne BA$

Since $AB$ means "B then A" and $BA$ means "A then B," they're generally different transformations — matrix multiplication is not commutative. Rotating a shape and then shearing it lands somewhere different from shearing then rotating. Drag through it below.

Why it matters Non-commutativity is not a nuisance — it's information. The order of layers in a neural network matters; the order of transformations in a graphics pipeline matters. "Which happens first" is a real modelling choice, and the algebra respects it.

Section 4Shapes, the identity, and associativity

The shape rule. You can multiply $A\,(m\times n)$ by $B\,(p\times q)$ only when the inner dimensions match — $n = p$ — and the result is $m\times q$:

$$ (m\times \underbrace{n)\,(n}_{\text{must match}}\times q) \;=\; m\times q $$

This is just the composition rule in disguise: $B$ outputs vectors in $\mathbb{R}^p$, and $A$ must accept inputs from $\mathbb{R}^n$ — so $n=p$ or the pipeline doesn't connect. It's also why $AB$ can be valid while $BA$ is undefined (shapes don't line up the other way).

Identity. The identity matrix $I$ (1s on the diagonal, 0s elsewhere) is the "do nothing" transformation: $AI = IA = A$.

Associativity. Grouping doesn't matter — $A(BC) = (AB)C$ — because both mean "do C, then B, then A." This is quietly essential: it's why you can precompute $AB$ once and reuse it, and why a deep network's layers can be reasoned about in chunks.

Going deeper — associative but not commutative

These two properties are easy to confuse. Associative ($A(BC)=(AB)C$) is about grouping — it always holds, because the sequence of operations is unchanged. Commutative ($AB=BA$) is about reordering — it generally fails, because swapping changes which transformation happens first. Function composition is the same: $f\circ(g\circ h) = (f\circ g)\circ h$ always, but $f\circ g \ne g\circ f$ in general. See MML §2.2.

Section 5In NumPy

import numpy as np

A = np.array([[1, 2], [0, 1]], dtype=float)
B = np.array([[2, 0], [1, 3]], dtype=float)

print(A @ B)                    # [[4. 6.] [1. 3.]]  "B then A"
print(B @ A)                    # [[2. 4.] [1. 5.]]  different! not commutative
print(np.allclose(A@B, B@A))    # False

# shape rule: (2x3)(3x4) -> (2x4); inner dims must match
M = np.random.rand(2, 3); N = np.random.rand(3, 4)
print((M @ N).shape)            # (2, 4)
[[4. 6.] [1. 3.]] [[2. 4.] [1. 5.]] False (2, 4)
Use @ (or np.matmul), never * In NumPy, * is element-wise multiplication (the Hadamard product), not matrix multiplication. Using * when you meant @ is one of the most common — and silent, when shapes happen to broadcast — bugs in ML code. Matrix product is always @ or np.matmul.

Section 6Where it lives in real ML

  • A neural-network forward pass is a chain of matrix products: $\mathbf{h} = f_2(W_2\,f_1(W_1\mathbf{x}))$. Stacking layers = composing transformations, exactly this session.
  • Batching — instead of one input vector, pass a matrix $X$ whose rows are 10,000 examples; $XW^\top$ scores them all at once. One matmul = a whole batch. This is why GPUs (matrix-multiply machines) dominate ML.
  • Attention — the relevance scores are $QK^\top$ (a matrix product of queries and keys), and the output multiplies those weights by $V$: two big matmuls per attention layer.
  • PCA / whitening / projections — chaining a rotation and a scaling is a single product you precompute once (associativity earning its keep).
The one-line takeaway Session 3: a matrix transforms space. Session 4: multiplying matrices composes those transformations. A deep network is just a long, learned composition — and the whole thing runs as matrix multiplications.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Right-to-left: $AB\mathbf{x}$ applies $B$ first. The matrix closest to the vector acts first.
  • Inner dimensions must match: $(m\times n)(n\times p)$ works; anything else is undefined.
  • $AB$ valid ≠ $BA$ valid: a $2\times3$ times a $3\times2$ gives $2\times2$; the reverse gives $3\times3$ — different, and sometimes only one direction is even legal.
  • $(AB)^\top = B^\top A^\top$ — transposing a product reverses the order (a frequent source of shape bugs).

Golden rules

  1. Matrix multiplication is composition: $AB$ = "do B, then A."
  2. Entry $(i,j)$ = row $i$ of A · column $j$ of B; equivalently, column $j$ of $AB$ = $A$ · (column $j$ of B).
  3. Not commutative ($AB\ne BA$) but associative ($A(BC)=(AB)C$).
  4. Shape: $(m\times n)(n\times p) = m\times p$ — inner dims must match.
  5. In code, matrix product is @; * is element-wise (a different operation).
Trap 1 · `*` vs `@` A * B multiplies element-wise; A @ B is the matrix product. If shapes let both run, you'll get a wrong answer with no error. Always @ for composition.
Trap 2 · Assuming AB = BA Reordering matrices silently changes the transformation. In backprop and attention, keeping the multiplication order (and transposes) correct is most of the battle.
Trap 3 · Transpose of a product $(AB)^\top = B^\top A^\top$, not $A^\top B^\top$. Forgetting to reverse the order breaks shapes in gradient derivations.

PracticeExercises

Hints only — solve in the paired practice.ipynb, computing each by hand first.

E1easy

Multiply $\begin{bmatrix}1&2\\3&4\end{bmatrix}\begin{bmatrix}0&1\\1&0\end{bmatrix}$ by hand, then verify. What did the right-hand matrix do?

Show solution

$\begin{bmatrix}2&1\\4&3\end{bmatrix}$. The right matrix $\begin{bmatrix}0&1\\1&0\end{bmatrix}$ swaps the columns (it's a swap/permutation applied first).

np.array([[1,2],[3,4]]) @ np.array([[0,1],[1,0]])  # [[2 1] [4 3]]
E2easy

For $A=\begin{bmatrix}2&0\\0&1\end{bmatrix}$, $B=\begin{bmatrix}1&1\\0&1\end{bmatrix}$, compute both $AB$ and $BA$ and confirm they differ.

Show solution

$AB=\begin{bmatrix}2&2\\0&1\end{bmatrix}$, $BA=\begin{bmatrix}2&1\\0&1\end{bmatrix}$ — different, so order matters.

E3medium

Which of these products are valid, and what shapes? (a) (4×3)(3×2), (b) (3×2)(3×2), (c) (1×5)(5×1), (d) (5×1)(1×5).

Show solution

(a) valid → 4×2. (b) invalid (inner 2≠3). (c) valid → 1×1 (a scalar = a dot product!). (d) valid → 5×5 (an outer product). Note (c) and (d) use the same vectors, opposite order, wildly different results.

E4medium

Verify associativity: pick random 2×2 matrices $A,B,C$ and check $A(BC) = (AB)C$ with np.allclose.

Show solution
A,B,C = (np.random.rand(2,2) for _ in range(3))
np.allclose(A @ (B @ C), (A @ B) @ C)   # True
E5hard

A rotation by 90° is $R=\begin{bmatrix}0&-1\\1&0\end{bmatrix}$. Compute $R^4$ (apply it four times). Explain the result geometrically.

Show solution
R = np.array([[0,-1],[1,0]])
np.linalg.matrix_power(R, 4)   # [[1 0] [0 1]] == I

Four 90° rotations = 360° = back to start, so $R^4 = I$. Composing a transformation with itself is just repeated multiplication — the seed of matrix powers and, later, eigenvalues (Session 6).

InterviewQuestions you should be able to answer

Q1 · What does multiplying two matrices represent?
Composition of their transformations: $AB$ means apply $B$ first, then $A$. Each entry is a dot product of a row of A with a column of B.
Q2 · Is matrix multiplication commutative? Associative?
Not commutative ($AB\ne BA$ in general — order changes which transform acts first) but associative ($A(BC)=(AB)C$ — grouping doesn't change the sequence).
Q3 · State the shape rule.
$(m\times n)(n\times p) = m\times p$; the inner dimensions must match. It's the composition rule: B's output space must equal A's input space.
Q4 · How does a neural network use matrix multiplication?
Each layer computes $W\mathbf{x}$ (or $XW^\top$ for a batch) — a matrix product — then a bias and nonlinearity. The forward pass is a composition of these; batching turns many inputs into one matmul, which is why GPUs are used.
Q5 · What is $(AB)^\top$?
$B^\top A^\top$ — transposing a product reverses the order. It shows up constantly in backprop shape-matching.
