Session 16 tested a proportion with a z-test — but real data rarely fits that mold. You usually don't know the true variance, your samples are small, your data is categorical, or you're comparing more than two groups. Each case has its own tool: the t-test for means with unknown variance, the χ² test for counts and categories, and ANOVA for many groups at once. This session is the working data scientist's testing toolbox.

Section 1The t-distribution — small samples, unknown variance

The z-test assumed you knew the population standard deviation $\sigma$. In practice you estimate it from the sample, and that estimate is itself noisy — especially with few data points. To account for that extra uncertainty, you use the t-distribution instead of the normal. It looks like a bell but with heavier tails: extreme values are more likely, so it's more cautious about declaring significance.

Its shape is set by the degrees of freedom (df $= n-1$ for one sample). Low df → fat tails (small, uncertain sample); as df grows, the t-distribution converges to the normal (by ~df = 30 they're nearly identical — the CLT again). Slide df below and watch it happen.

Your Power BI / SQL anchor Comparing this quarter's average deal size to last quarter's, on 12 deals each? That's a small-sample, unknown-variance comparison — a t-test, not a z-test. The t-distribution is what keeps you honest when you don't have thousands of rows.

Section 2t-tests — comparing means

Three flavours cover most "did the average change?" questions:

  • One-sample t-test — is a sample's mean different from a known value? ("Is average session time ≠ 5 min?")
  • Two-sample (independent) t-test — do two separate groups have different means? (control vs treatment — the A/B test for a continuous metric)
  • Paired t-test — same subjects measured twice (before/after); tests the mean difference. More powerful when it applies, because it removes between-subject variation.

Each computes a t-statistic (effect ÷ its standard error) and a p-value from the t-distribution. It assumes roughly normal data (or large enough $n$ via the CLT) and, for the two-sample version, comparable variances.

Section 3χ² tests — for counts and categories

t-tests are for means of continuous data. When your data is categorical counts, use the chi-square ($\chi^2$) test, which compares observed counts to expected ones:

$$ \chi^2 = \sum \frac{(\text{observed}-\text{expected})^2}{\text{expected}} $$
  • Goodness-of-fit — does one categorical variable match an expected distribution? ("Is this die fair?" — compare roll counts to uniform.)
  • Test of independence — are two categorical variables related? Build a contingency table (e.g. device × converted?) and test whether the rows and columns are independent.

Big χ² (observed far from expected) → small p → reject "they match / are independent."

Section 4ANOVA — comparing several groups at once

To compare the means of three or more groups, don't run a t-test on every pair — that's multiple comparisons and inflates false positives (Session 16). Instead use ANOVA (Analysis of Variance), which asks one question: "is there any difference among the group means?" via the F-statistic:

$$ F = \frac{\text{variance between groups}}{\text{variance within groups}} $$

If the groups truly differ, the between-group variance dwarfs the within-group noise, so $F$ is large and $p$ is small. A significant ANOVA says "the groups aren't all equal"; you then use post-hoc tests (with correction) to find which pairs differ.

Section 5In SciPy ✓ outputs verified

import numpy as np
from scipy import stats
rng = np.random.default_rng(1)

# two-sample t-test: control vs treatment
a = rng.normal(100, 15, 30); b = rng.normal(108, 15, 30)
t, p = stats.ttest_ind(a, b)
print(round(t,3), round(p,4))          # -2.505 0.0151  -> B significantly higher

# chi-square goodness of fit: is the die fair?
obs = np.array([16,18,16,14,22,14])     # 100 rolls
print(np.round(stats.chisquare(obs), 4))  # chi2=2.72, p=0.7431 -> fair (fail to reject)

# chi-square independence: device x converted?
table = np.array([[30,10],[20,40]])
chi2, pi, dof, _ = stats.chi2_contingency(table)
print(round(chi2,3), round(pi,5))     # 15.042 0.00011 -> dependent

# ANOVA: three groups
g1=rng.normal(50,5,20); g2=rng.normal(52,5,20); g3=rng.normal(58,5,20)
print(np.round(stats.f_oneway(g1,g2,g3), 4))  # F=24.55, p~0 -> groups differ
-2.505 0.0151 [2.72 0.7431] 15.042 0.00011 [24.5543 0. ]
Assumptions still matter These tests assume (roughly) normal data / large samples, and — for two-sample t and ANOVA — comparable variances. Heavy skew, outliers, or wildly unequal spreads call for variants (Welch's t-test, which SciPy uses with equal_var=False) or non-parametric tests (Mann–Whitney, Kruskal–Wallis). Check before you trust the p-value.

Section 6Where it lives in real ML

  • A/B tests on continuous metrics (time-on-page, revenue-per-user) use a t-test; on rates, the z-test (Session 16).
  • χ² independence screens whether a categorical feature relates to the target — a quick feature-relevance check.
  • ANOVA compares more than two model variants or segments in one shot, controlling false positives vs many pairwise tests.
  • Model comparison across CV folds is often a paired t-test (same folds, two models) — more powerful because it removes fold-to-fold variation.
The one-line takeaway Match the test to the data: t for means (small samples / unknown variance), χ² for categorical counts, ANOVA for many groups. All produce a statistic and a p-value read exactly as in Session 16 — just with the right sampling distribution.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Use t, not z, for small samples with estimated variance — the fat tails guard against over-confidence.
  • Paired vs independent — using an independent t-test on paired data throws away power (and can mislead).
  • χ² needs adequate expected counts (rule of thumb ≥5 per cell); tiny counts need Fisher's exact test.
  • ANOVA finds a difference, not which — follow with corrected post-hoc comparisons.

Golden rules

  1. Unknown variance + small $n$ → the t-distribution (df = $n-1$); → normal as df grows.
  2. t-tests compare means: one-sample, two-sample (A/B), paired (before/after).
  3. χ² for categorical counts: goodness-of-fit and independence (contingency tables).
  4. ANOVA (F-test) compares ≥3 group means at once — avoid many pairwise t-tests.
  5. Every test → a statistic + a p-value; check assumptions before trusting it.
Trap 1 · z-test on a tiny sample With few points and an estimated σ, the normal is over-confident. Use the t-test; its heavier tails demand stronger evidence.
Trap 2 · Many pairwise t-tests instead of ANOVA Comparing k groups pairwise multiplies false positives. Run ANOVA first; only if significant, do corrected post-hoc pairs.
Trap 3 · Ignoring test assumptions Unequal variances, heavy skew, or paired structure change which test is valid. Use Welch's t, non-parametric tests, or the paired version as appropriate.

InterviewQuestions you should be able to answer

Q1 · When do you use a t-test instead of a z-test?
When the population variance is unknown (estimated from the sample) and/or the sample is small. The t-distribution's heavier tails account for the extra uncertainty; it converges to the normal as df grows.
Q2 · One-sample vs two-sample vs paired t-test?
One-sample: mean vs a known value. Two-sample: two independent groups' means. Paired: same subjects measured twice (tests the mean difference) — more powerful when applicable.
Q3 · What is a χ² test for?
Categorical counts: goodness-of-fit (does one variable match an expected distribution) and independence (are two categorical variables related, via a contingency table).
Q4 · Why ANOVA instead of many t-tests?
To compare ≥3 groups while controlling the false-positive rate. Many pairwise tests inflate Type I error; ANOVA's single F-test asks "are any means different?" first.
Q5 · How do you compare two models across CV folds?
A paired t-test on the per-fold scores (same folds, two models) — pairing removes fold-to-fold variance, increasing power.
