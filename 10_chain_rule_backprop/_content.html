A neural network is a function stuffed inside a function stuffed inside a function — layer after layer, each feeding the next. To train it you need the derivative of the final loss with respect to every weight buried deep inside that nesting. The tool that gets it is the chain rule: the derivative of a composition is the product of the derivatives. Run that rule backward through the layers and you have backpropagation — the single algorithm that trains essentially every deep model on earth.

This is the payoff of the whole calculus pillar. Sessions 8–9 built the derivative and the gradient; this session composes them into the training algorithm itself.

By the end you will be able to
  • Apply the chain rule: differentiate $f(g(x))$ as $f'(g(x))\cdot g'(x)$.
  • Read a computational graph and run its forward pass.
  • Run backprop: each node passes back (local gradient × upstream gradient).
  • Explain why reverse mode is efficient for one loss and many parameters.
  • Recognize vanishing / exploding gradients as a product of many derivatives.

Section 1The chain rule 📖 MML §5.2.2

If a quantity passes through two functions in a row — $g$ then $f$ — its overall rate of change is the product of the two rates:

$$ \frac{d}{dx}\,f\big(g(x)\big) = f'\big(g(x)\big)\cdot g'(x) $$

Intuition: if $g$ changes 3× as fast as $x$, and $f$ changes 2× as fast as $g$, then $f$ changes $3\times2=6\times$ as fast as $x$. Rates multiply along a chain. For example, $\frac{d}{dx}\sin(x^2) = \cos(x^2)\cdot 2x$ — the derivative of the outer ($\cos$) times the derivative of the inner ($2x$). For many nested functions you just multiply all the links:

$$ \frac{dL}{dx} = \frac{dL}{dh}\cdot\frac{dh}{dz}\cdot\frac{dz}{dx} $$
Your Power BI / SQL anchor It's unit conversion. Miles→km is ×1.61; km→m is ×1000; so miles→m is ×1.61×1000. Each "how much does the next thing change per unit of the previous" multiplies down the chain. Backprop is that same multiplication, applied to a network's layers.

Section 2The computational graph & forward pass

Any model is a computational graph: inputs and weights flow through a sequence of simple operations to produce the loss. The forward pass just evaluates that graph left-to-right, storing each intermediate value. Take a single neuron with weight $w$, input $x$, target $y$:

$$ z = w\,x \;\to\; a = \sigma(z) \;\to\; L = (a-y)^2 $$

Drag the weight below to run the forward pass, then hit Backprop to watch the gradient flow back through the same graph.

Section 3Backprop = the chain rule, run backward 📖 MML §5.6

Backpropagation is the chain rule applied to the graph from the output back to the inputs. The rule at every node is identical and beautifully local:

$$ \underbrace{\frac{\partial L}{\partial(\text{node input})}}_{\text{gradient to pass back}} = \underbrace{\frac{\partial L}{\partial(\text{node output})}}_{\text{upstream gradient}} \times \underbrace{\frac{\partial(\text{node output})}{\partial(\text{node input})}}_{\text{local gradient}} $$

For our neuron, working right-to-left:

$$ \frac{dL}{da}=2(a-y) \;\to\; \frac{dL}{dz}=\frac{dL}{da}\cdot\underbrace{a(1-a)}_{\sigma'(z)} \;\to\; \frac{dL}{dw}=\frac{dL}{dz}\cdot\underbrace{x}_{\partial z/\partial w} $$

Each node only needs its local derivative and the gradient handed to it from downstream — it never needs to know about the rest of the network. That locality is what makes backprop scale to billions of parameters: it's the same three-line pattern, repeated.

Section 4Why reverse mode — and vanishing gradients

You could apply the chain rule forward (input→output) or backward (output→input). For ML the choice is decisive: we have one scalar loss and millions of parameters. Reverse mode computes the loss's gradient w.r.t. all parameters in a single backward pass; forward mode would need one pass per parameter. That efficiency is the whole reason backprop (reverse-mode autodiff) won.

Going deeper — forward vs reverse, and why gradients vanish/explode

Forward mode pushes derivatives input→output, cheap when there are few inputs and many outputs. Reverse mode pushes output→input, cheap for many inputs and few outputs — exactly the ML shape (many weights, one loss). Cost of reverse mode ≈ one forward pass, for all gradients at once.

But because the chain rule multiplies a derivative per layer, a deep network's gradient is a long product. If those local derivatives are mostly < 1, the product shrinks toward zero — vanishing gradients (early layers barely learn); if mostly > 1, it blows up — exploding gradients. Sigmoid's derivative peaks at just 0.25, so deep sigmoid stacks vanish — which is why ReLU (derivative 1 for positive inputs), careful initialization, normalization, and residual/skip connections were invented. See MML §5.6.

Section 5In NumPy — backprop by hand ✓ outputs verified

import numpy as np
sig = lambda z: 1/(1+np.exp(-z))

x, y, w = 1.5, 1.0, 0.5

# forward pass — store intermediates
z = w*x
a = sig(z)
L = (a - y)**2

# backward pass — chain rule, right to left
dL_da = 2*(a - y)          # dL/da
dL_dz = dL_da * a*(1-a)     # × local σ'(z) = a(1-a)
dL_dw = dL_dz * x            # × local ∂z/∂w = x
print(L, dL_dw)

# check against a finite-difference gradient
g = lambda w: (sig((w)*x) - y)**2
print((g(w+1e-6) - g(w-1e-6)) / (2e-6))   # ≈ dL_dw ✓
0.1029 -0.2097 -0.2097 # finite-difference matches the hand-derived gradient
Frameworks do this automatically — but know the shapes loss.backward() in PyTorch runs exactly this backward sweep over the whole graph, filling in .grad for every parameter. It uses Jacobians (Session 9) at each node and multiplies them (reverse order, so $(AB)^\top=B^\top A^\top$ from Session 4 shows up). Understanding the by-hand version is what lets you debug when a gradient is None or a shape is wrong.

Section 6Where it lives in real ML

  • Every training step is forward pass → loss → backward() (backprop) → optimizer step. This session is that middle arrow.
  • Autodiff frameworks (PyTorch, JAX, TensorFlow) build the computational graph and run reverse-mode backprop for you — the reason modern deep learning is feasible.
  • Activation & architecture choices are largely about keeping the gradient product healthy: ReLU/GELU, BatchNorm/LayerNorm, residual connections all fight vanishing/exploding gradients.
  • Gradient clipping caps exploding gradients (common in RNNs/transformers) — a direct patch on the "product blows up" problem.
The one-line takeaway The chain rule multiplies rates along a chain; backprop is that rule run backward through the computational graph, so one pass yields every parameter's gradient. It's the algorithm that makes training possible — and its multiplicative nature is why deep nets need ReLU, normalization, and skip connections.

ReferenceEdge cases, rules & traps

Edge cases & gotchas

  • Store the forward values. Backprop needs the intermediates (e.g. $a$ for $\sigma'=a(1-a)$); frameworks cache them, which is why the graph uses memory.
  • Order matters (Session 4). Backward multiplies Jacobians in reverse; transposes appear ($(AB)^\top=B^\top A^\top$).
  • Non-differentiable nodes (ReLU at 0, max, argmax) use subgradients or block the gradient — know where your graph is smooth.
  • Detaching / stop-gradient cuts an edge in the backward graph on purpose (e.g. targets in RL, teacher networks).

Golden rules

  1. Chain rule: derivative of a composition = product of derivatives.
  2. Backprop at each node: gradient-in = upstream gradient × local gradient.
  3. It runs backward through the graph; one pass → all parameter gradients.
  4. Reverse mode is used because there's one loss and many parameters.
  5. A long product of derivatives ⇒ vanishing/exploding gradients — mitigated by ReLU, norm, residuals, clipping.
Trap 1 · Forgetting the local × upstream structure Every node's returned gradient is upstream × its own local derivative. Skipping the local factor (or the upstream one) is the classic hand-backprop bug.
Trap 2 · Sigmoid/tanh in deep stacks Their derivatives are ≤ 0.25 / ≤ 1, so many layers multiply toward zero → vanishing gradients and dead training. Prefer ReLU-family + normalization for depth.
Trap 3 · Reusing a graph after backward Frameworks free the graph after backward() by default; calling it twice errors unless you retain_graph. And forgetting optimizer.zero_grad() accumulates stale gradients.

Hints only — solve in the paired practice.ipynb.

E1easy

Differentiate $f(x)=\sin(x^2)$ and $g(x)=e^{3x}$ with the chain rule.

Show solution

$f'(x)=\cos(x^2)\cdot2x$; $g'(x)=e^{3x}\cdot3=3e^{3x}$.

E2easy

For $L=(a-y)^2$ with $a=\sigma(z)$, write $dL/dz$ using the chain rule.

Show solution

$\frac{dL}{dz}=\frac{dL}{da}\cdot\frac{da}{dz}=2(a-y)\cdot a(1-a)$.

E3medium

Implement the single-neuron forward+backward from Section 5 and match $dL/dw$ against a finite difference.

Show solution
sig=lambda z:1/(1+np.exp(-z)); x,y,w=1.5,1.0,0.5
z=w*x; a=sig(z)
dL_dw=2*(a-y)*a*(1-a)*x
g=lambda w:(sig(w*x)-y)**2
np.isclose(dL_dw,(g(w+1e-6)-g(w-1e-6))/2e-6)   # True
E4medium

Two-layer chain: $z_1=w_1x$, $a=\sigma(z_1)$, $z_2=w_2a$, $L=(z_2-y)^2$. Find $dL/dw_1$ by the chain rule.

Show solution

$\frac{dL}{dw_1}=\underbrace{2(z_2-y)}_{dL/dz_2}\cdot\underbrace{w_2}_{dz_2/da}\cdot\underbrace{a(1-a)}_{da/dz_1}\cdot\underbrace{x}_{dz_1/dw_1}$ — four links multiplied.

E5hard

Show numerically why deep sigmoid stacks vanish: multiply 10 sigmoid derivatives $a(1-a)$ (with $a\approx0.5$) and compare to multiplying 10 ReLU derivatives (=1).

Show solution
print(0.25**10)   # ~9.5e-7  -> vanished
print(1.0**10)    # 1.0      -> preserved

Sigmoid's derivative maxes at 0.25, so ten layers multiply to ~$10^{-6}$ — the early-layer gradient all but disappears. ReLU's derivative of 1 keeps the signal alive. This is the mathematical case for ReLU + residual connections.

InterviewQuestions you should be able to answer

Q1 · State the chain rule and give an example.
$\frac{d}{dx}f(g(x))=f'(g(x))g'(x)$ — derivatives multiply along a composition. E.g. $\frac{d}{dx}\sin(x^2)=\cos(x^2)\cdot2x$.
Q2 · What is backpropagation?
The chain rule applied backward through a computational graph: each node returns (upstream gradient × its local derivative), so one backward pass yields the loss gradient for every parameter.
Q3 · Why reverse mode instead of forward mode?
ML has one scalar loss and many parameters. Reverse mode gets all parameter gradients in a single pass (~one forward pass of cost); forward mode would need a pass per input/parameter.
Q4 · What causes vanishing/exploding gradients?
The gradient is a product of one derivative per layer. Derivatives consistently <1 shrink it to zero (vanishing); >1 blow it up (exploding). Fixes: ReLU, normalization, residual connections, gradient clipping.
Q5 · Why must the forward pass store intermediate activations?
The backward pass needs them to compute local derivatives (e.g. $\sigma'=a(1-a)$ needs $a$). That's why training uses much more memory than inference.
